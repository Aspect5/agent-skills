# Coverage report template

Load when writing the deliverable (Workflow Step 7). The report is a **coverage
read**, not a victory lap: it states what is verified, what is **not**
(`base:coverage-honest`), and the exact command to run the suite. It never
fabricates a passing result (`base:report-no-fabrication`) — every score shown is
from a real run, or the case is labelled `pending-labels` / `needs-human`.

Write the full report to a file (default `eval-coverage.md` in the repo root, or a
profile/user-named path), **incrementally**. Emit only a tight summary inline (the
tally + the single biggest gap + the run command + the path).

```markdown
# Verification coverage — <repo / surface label>

**Run it:** `<the exact command, e.g. make eval / pytest evals/ -m eval>`
**Headline:** <one sentence — the biggest remaining gap, stated plainly.>
**Tally:** deterministic surfaces <V/T verified> · non-deterministic surfaces
<V/T verified> · evals scaffolded: <N output, M trajectory> · datasets:
<K cases, B buckets> · LM-judge agreement: <X% on N cases | n/a — no judge>

## Coverage matrix (the denominator is every surface, not just touched files)

| Surface | Kind | Verification now | Status | Where |
|---|---|---|---|---|
| `path/to/parser.py::parse` | deterministic | unit test, boundary+error | added | `tests/test_parser.py` |
| `path/to/router.py::route` | deterministic | — | unverified | (gap — see below) |
| `agents/tutor.py` answer | non-det (output) | rubric + LM-judge, 30 cases | scaffolded | `evals/tutor/` |
| `agents/tutor.py` trajectory | non-det (traj) | tool-path + budget check | partial | `evals/tutor/` (safety axis TODO) |
| `agents/planner.py` | non-det (output) | — | unverified | (gap — see below) |

Legend: verified/scaffolded · partial (axis or bucket missing) · unverified.

## What is NOT verified (state it as plainly as the wins)
- `router.py::route` — deterministic, no test. Cheap to add; one input→output table.
- `planner.py` trajectory — no eval; the multi-step plan is unscored. Highest risk.
- Adversarial bucket on `tutor` is thin (3 cases) — safety is under-measured.

## What was scaffolded
### Deterministic tests
- `tests/test_parser.py` — N cases (happy + boundary + error). Demonstrated
  failing case: <the known-bad input that fails on the bug, passes on the fix>.

### Evals
- `evals/tutor/` — output eval, 30 labelled cases across <buckets>. Rubric:
  `evals/tutor/rubric.md` (criteria: correctness, faithfulness, refusal). Judge:
  `<model>`, calibrated to <N> human labels → **<X>% agreement**. Threshold:
  <floor>; **block / report-only**: <which>.
  - **Vacuity check:** ran on a seeded known-bad output → **failed on
    <criterion>** as designed (proof it can fail).
- `evals/tutor/` — trajectory eval: tool-selection + step-budget checks
  (programmatic over recorded traces). Safety-invariant axis: <done | TODO>.

## Real run results (no fabrication)
<Paste the actual run output: per-bucket pass rates + aggregate + the literal
exit/threshold line. If labels are incomplete, list those cases as
`pending-labels` — do NOT show a green score you did not produce.>

## How to run & extend
- Full suite: `<command>` · Cheap PR subset: `<command>` · Gate tier:
  <per-PR | nightly | pre-release | on-label> (cost posture: <why>).
- Add a regression case: drop a row in `evals/<suite>/dataset/cases.jsonl` with a
  label + rationale (every production miss should become a case).
- Re-calibrate the judge when the model/rubric changes
  (`evals/<suite>/calibration.jsonl`).

## Handoffs
- Specification gaps surfaced while building the dataset (cases where "correct" was
  undefined): <list — these need a spec, see spec-author>.
- Harness/config risks noticed (wrong tool exposed, missing guardrail): <list —
  see harness-audit>.

_Generated by the eval-author skill. It scaffolds verification and reports
coverage; it does not invent passing results. Labels and thresholds need human
sign-off before this gate is trusted._
```

## Inline summary (what actually goes in chat)

Keep it to <=6 lines:

- The **tally** line (deterministic V/T, non-det V/T, judge agreement).
- The **single biggest gap** (the one a human should fix next).
- The **run command**.
- The **path to the full report**.

Never paste the full matrix or the dataset into chat — that blows the output-token
budget the skill exists to protect.
